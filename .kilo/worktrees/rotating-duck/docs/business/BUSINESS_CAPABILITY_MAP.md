## 1. CAPABILITY FRAMEWORK OVERVIEW

### 1.1 What is a Capability?

```
┌─────────────────────────────────────────────────────────────┐
│  CAPABILITY = "What We Must Be Able To Do"                  │
│                                                             │
│  Capability ≠ Activity                                       │
│  Capability = Ability to perform reliably at scale          │
│                                                             │
│  Example:                                                   │
│  ├── Activity: "Send email to customer"                    │
│  └── Capability: "Manage customer lifecycle at 10K+/month"  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 Capability Map Structure

```
┌─────────────────────────────────────────────────────────────┐
│  YALIHAN CAPABILITY MAP — 4-Level Hierarchy               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  LEVEL 1: DOMAIN (Strategic Areas)                         │
│  ├── Customer & Sales                                       │
│  ├── Operations                                            │
│  ├── Finance & Risk                                        │
│  └── Technology & Innovation                               │
│                                                             │
│  LEVEL 2: CAPABILITY (What We Do)                          │
│  ├── Domain: Customer & Sales                               │
│  │   ├── Lead Management                                   │
│  │   ├── Property Valuation                               │

```
